* Java ee : 웹 개발용 -> perspective 추가하고 변경해서 조정 가능(디폴트로는 java사용)
* 실행파일 하나에 여러개의 .java 파일들로 구성 -> 그 파일들을 모아서 프로그램으로 만든 단위가 프로젝트
* 수업 날짜 기준 JavaSE-1.8이 대세 -> 각 회사별로 어떤 버전을 쓰는지가 중요
* 사람이 작성한 코드는 .java -> src 파일에
  컴퓨터가 이해할 수 있는 번역된 코드는 .class -> bin 파일에
 ∴ 과제 폴더는 src에 있는걸 올려야 됨

* JAVA를 정의할때 상속과 다용성을 정의할 줄 알아야 됨
-> java를 알고나면 디자인 패턴 공부를 해볼것(중요!!)
-> 디자인 패턴, 객체 지향 방법론 관련 책들

* WORA(Write Once Run Anywhere) -> 어떤 os든지 활용 가능(플랫폼 독립적)
  JVM이라는 자체적 os를 가지기 때문

* 컴파일 : 컴퓨터가 이해할 수 있는 코드로 변환하는 것
  ~.java -> 컴파일 -> ~.class -> JVM이 이를 해석(interprete)	해서 출력

* 프로그램의 시작과 끝을 알려주는 명령어 : main method

* javac Hello.java -> Hello.java파일을 컴파일 해줘라는 명령어(javac) -> 이러한 작업들을 도와주는 tool 중 하나가 eclipse
