생성자에서의 변수를 사용하는 경우
super -> 부모의 것을 호출해와서 쓰는것이기 때문에 setter를 해줄 필요가 없음

묵시적 형변환이 되어있어야지만 : 자식의 공간(new)을 만들고 부모가 이를 카르킴 
명시적 형변환이 가능함


묵시적 형변환을 통해서 부모클래스명으로 호출을 할 수 있게끔 하여 자식 클래스들이 
명칭이 바뀌어도 호출하는 쪽에서 문제가 없게끔 하는 것
-> 단 부모클래스에서 메소드를 추상화 시켜서 자식 클래스가 메소드들을 오버라이드 할 수 있게끔 해야됨





static 안붙어있는 메소드는 객체가 만들어져야지만 사용가능
그래서 static이 붙어있는 메소드에서 해당 클래스 내에 있는 일반 메소드를 호출하는것이 불가능
-> this.일반 메소드 혹은 멤버변수 사용 불가능
하지만 static이 붙어있는 메소드내에서 일반 객체를 생성하는 것은 가능
->
static void print(){
	AAA a = new AAA();
	a.info();
}
