추상은 클래스랑 메소드에만 존재
추상 클래스는 -> 유지,보수를 위해서 존재함

추상클래스 -> 클래스 내부에 1이상의 추상메소드를 가지고 있는 클래스 : 추상메소드가 하나라도 있으면 추상 클래스로 생성
-> 이미 완성되어있는 일반 메소드가 존재할 수도 있음
추상클래스는 그래서 인스 객체를 만들수 없음
추상메소드 -> 메소드 생성에 있어서 강제성을 부여하기 위해 사용(정해진 메소드의 이름과 갯수를 강제)

abstract 메소드명() -> 메소드의 정의는 있지만 내용은 없음
-> 이런이런 이름의 메소드가 필요합니다만 정의할 뿐, 메소드 내용은 없음

추상메소드는 상속과 관련이 있음 -> 해당 메소드를 자식 클래스에서 강제성을 가지고 생성하게 만듬
즉 추상클래스를 상속받은 하위클래스는 추상메소드를 오버라이딩 해야됨 

부모 클래스에서 추상메소드를 만들어서 강제성을 가지고 자식 메소드에서 해당 메소드를 생성하게 함
