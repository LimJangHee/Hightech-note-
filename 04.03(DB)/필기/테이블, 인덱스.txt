< 기본 단어 >
테이블 : 데이터 저장하는 객체, RDB 기본 단위
- sql로 테이블에서 원하는 데이터 꺼내 올 수 있음
- 특정 주제, 목적으로 만들어 지는 집합

인덱스 : 검색을 빨리 하기위해서 사용하는 오브젝트(주소값) -> 존재 이유
테이블과 1:1로 매핑
테이블 1개에 여러개의 인덱스를 가질 수 있음

기본키 : 테이블에 존재하는 각 행을 한가지 의미로 특정할 수 있는 한 개 이상의 컬럼들(식별자)
-> 중복을 방지함 
ex) 홍길동이라는 사람이름과 주소의 송장번호를 기본키 하나로 지정하여 사용
상품들의 이름을 기본키로 각각 지정하여 필요할때마다 불러와서 사용 -> 상품 생성될때마다 생성할 필요가 없어짐

< 데이터 유형 > 
* CHAR(n) : 고정길이 문자
* VARCHAR2(n)  : 가변길이 문자 -> 지정한 범위 내애서 가변적으로 사용 가능 : 주로 사용하게 될것
* NUMBER(	p,s) : 가변길이 숫자
- p : 소수점을 포함한 전체 자리수
- s : 소수점 자릿수를 의미
ex) 5,2 -> 정수 3자리, 소수 2자리
* DATE : 날짜 -> 년 ~ 초
(* TIMESTAMP -> 년 ~ 초 + 밀리초까지)

< 데이터 검색 방법 >
* FULL TABLE SCAN : 전체 데이터를 읽으면서 조건 부합 레코드 추출 -> 비효율적
* INDEX SCAN TABLE READ : 인덱스 조건에 부합하는 레코드를 찾음 -> 효율적

< 테이블 생성 >
테이블스페이스 생성(DISK에) + 어떤 데이터 타입으로 만들지 지정
-> 블록 단위로 데이터를 채워 넣음 : 인덱스를 나눌시 인덱스별로 블록을 나눠서 채워넣음
