파티셔닝은 병렬 데이터베이스 환경 중 수평분할에서 활용되는 분할 기법이다. 파티셔닝의 유형에는 range, hash, list, composite, round robin이 있다

병행제어 기법의 종류
로킹 기법
타임 스탬프 기법
최적 병행 수행 기법 (검증 기법, 확인 기법, 낙관적 기법)
다중 버전 기법

이상 현상
삽입, 삭제, 갱신

물리적 데이터베이스 설계 -> 인터페이스 설계는 논리적임

create table문에 포함 x -> 속성 타입 변경

분산 데이터베이스의 구성 요소
분산 처리기, 분산 데이터베이스, 통신 네트워크, 분산 트랜잭션

데이터베이스 병행제어 목적
-> 데이터베이스 공유 최대화(최소화 아님)

물리적 데이터베이스 구조 설계 -> 트랜잭션 모델링은 해당 안됨(개념적)

데이터베이스 설계단계에서의 트랜잭션 설계단계
- 개념설계 : 트랜잭션 모델링
- <논리설계 : 트랜잭션 인터페이스 설계>
- 물리설계 : 트랜잭션 세부설계
